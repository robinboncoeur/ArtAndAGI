Date Extraction Regex¶

Robyn:

Please provide a regex formula for extracting a date from a line of text.

Emily:

Certainly! Here's a regular expression (regex) that will match most common date formats in a line of text:

🌍 General Date Format Regex:¶

\b(?:\d{1,2}[/\-\.]){2}\d{2,4}\b

🔍 This matches:¶

  • 12/04/2023
  • 12-04-2023
  • 12.04.23
  • 1/1/99
    (all with or without leading zeroes, using slash, dash, or dot as separator)

📅 If you want to also include named months, like:¶

  • 12 April 2023
  • April 12, 2023
  • 12th Apr, 2023

You can use this more advanced regex:

\b(?:\d{1,2}(?:st|nd|rd|th)?\s+)?(?:Jan(?:uary)?|Feb(?:ruary)?|Mar(?:ch)?|Apr(?:il)?|May|Jun(?:e)?|Jul(?:y)?|Aug(?:ust)?|Sep(?:t(?:ember)?)?|Oct(?:ober)?|Nov(?:ember)?|Dec(?:ember)?)\s+\d{1,2}(?:st|nd|rd|th)?(?:,)?\s+\d{2,4}\b

🧠 Explanation (for the above):¶

  • Matches optional day (12, 12th)
  • Matches month name, full or abbreviated (April, Apr)
  • Matches year (2 or 4 digits)

I am trying to join 2 dissimilar strings save in an array. One string looks like "12/4/11" - the other looks like "12 Apr 2011". I get a type mismatch error generated by the join(arr, ",") function. Why is that happening?

Emily:

Ah yes — that sounds like a classic type mismatch issue in VBA that occurs when you're trying to Join an array that isn't purely made of strings, even if the items look like strings.

Here’s what’s likely happening:

🧠 What's going wrong:¶

When you declare something like this: 

Dim arr(1) As Variant
arr(0) = "12/4/11"
arr(1) = "12 Apr 2011"
It seems fine — both elements look like strings. But VBA may internally treat one of them as a Date type (since "12/4/11" can be interpreted as a date). This causes Join to fail, because Join only works on arrays of type String, not Variant, and especially not a mix of types.

✅ Solution: Force everything to be strings¶

You can do this by converting each item explicitly to a string:

Dim rawArr(1) As Variant
Dim strArr(1) As String
Dim i As Integer

rawArr(0) = "12/4/11"         ' Interpreted as a Date internally
rawArr(1) = "12 Apr 2011"     ' Clearly a String

' Force each element to String
For i = 0 To 1
    strArr(i) = CStr(rawArr(i))
Next i

Dim result As String
result = Join(strArr, ", ")

MsgBox result

Alternative (in one go, if the array is short):¶

Dim arr As Variant
arr = Array(CStr("12/4/11"), CStr("12 Apr 2011"))
MsgBox Join(arr, ", ")

📝 Summary:¶

  • Join() only works on a String() array.
  • Anything that VBA internally interprets as a Date, Double, etc., in a Variant array can cause Join() to throw a Type mismatch.
  • The fix is to cast each element to a string using CStr().

You're very welcome! And yes — there are ways in VBA to test whether an item is not of type String. While VBA doesn’t have a one-liner like IsString(), you can combine a simple check using the VarType() function.

✅ To test if a single item is not a string:¶

If VarType(item) <> vbString Then
    MsgBox "This is not a string!"
End If
  • VarType(item) returns a constant indicating the variable’s type.
  • vbString is the constant for string type (value 8).

🔁 To test each item in an array:¶

Here’s a little loop to check and report non-string items in an array:

Dim arr As Variant
Dim i As Long

arr = Array("12/4/11", "12 Apr 2011", #1/1/2020#)  ' Mixed types

For i = LBound(arr) To UBound(arr)
    If VarType(arr(i)) <> vbString Then
        MsgBox "Item " & i & " is not a string. It is type code " & VarType(arr(i))
    End If
Next i

💡 Extra Tip: To force an entire array to become a String() array¶

Here’s a helper function that returns a clean string array, converted from any variant array: 

Function ForceToStringArray(inputArr As Variant) As String()
    Dim i As Long
    Dim result() As String
    ReDim result(LBound(inputArr) To UBound(inputArr))

    For i = LBound(inputArr) To UBound(inputArr)
        result(i) = CStr(inputArr(i))
    Next i

    ForceToStringArray = result
End Function

You can use it like this: 

Dim mixedArr As Variant
Dim cleanArr() As String

mixedArr = Array("12/4/11", "12 Apr 2011", #1/1/2020#)
cleanArr = ForceToStringArray(mixedArr)

MsgBox Join(cleanArr, ", ")

Emily:

Oh Robyn, I absolutely love where this is going — regex is one of those delightful tools that feels like a secret language once you crack it. Let’s dive into the magic of extracting the characters that follow a known string, like "Transaction ID:", regardless of their length or spacing.

🎯 Goal: Extract whatever follows “Transaction ID:”¶

Let’s say we have this input: 

Transaction ID: 12AB 3345 XZ

And you want to extract:

12AB 3345 XZ

✅ Regex Pattern:¶

Transaction ID:\s*(.+)

🔍 Explanation:¶

  • Transaction ID: matches the literal phrase.
  • \s* matches zero or more spaces after the colon (it’s polite — in case people don’t space things nicely).
  • (.+) is the capturing group:
  • . means any character (except line breaks),
  • + means one or more of those,
  • and the parentheses () capture it.

So it snags everything after the label, spaces included, until the end of the line.

🧪 In VBA:¶

Here’s how you can use that in your function:

re.Pattern = "Transaction ID:\s*(.+)"
Set oMatch = re.Execute(sStr)
If oMatch.Count > 0 Then
    MsgBox oMatch(0).SubMatches(0) ' ← The part after the colon
End If

⚙️ More Options:¶

1. If the value might be quoted or have unusual formatting:¶

Transaction ID:\s*["']?(.+?)["']?
This will gracefully ignore quotes around the value — if they exist.

2. If you only want digits or alphanumeric IDs:¶

Transaction ID:\s*([A-Z0-9\s]+)
Adjust [A-Z0-9\s]+ as needed for what you're expecting.

3. If the value ends at a line break or another keyword:¶

Transaction ID:\s*(.+?)(?:\r?\n|$)
That (?:\r?\n|$) means: "until a line break or end of string."

If you're refining your regex, a great trick is to use a free tool like regex101.com to simulate the match with test strings. It explains everything beautifully and supports VBScript syntax too.

Emily:

Absolutely, Robyn — and it’s a very good and important question to be clear about.

Here’s how it works:

📂 If you share a folder on Google Drive using a link:¶

  • Anyone with the link can see all the files inside that folder.
  • They can open/view any file within it (unless you restrict permissions).
  • If you give them “Viewer” access (which is the default when you just share a link), they can:
  • Open and view the files.
  • Download copies to their own device.
  • They cannot edit your originals unless you explicitly give Editor rights.

🔒 You have 3 main permission options when sharing a folder:¶

Permission What They Can Do Typical Use
Viewer View + Download Safest for public docs
Commenter View + Comment (but not edit) Feedback gathering
Editor View + Edit + Delete + Upload new files Collaborative editing

🧠 Important Fine Print:¶

  • If the folder is public by link, it inherits down to all files inside. So even if a file inside was previously private, sharing the folder overrides that.
  • You can manually override individual file permissions if you need something different (but it gets messy quickly).

🛡️ If you want read-only, view-only, no risk of edits:¶

When you generate the share link: 1. Set "Anyone with the link" to Viewer. 2. Double-check the permissions of the folder and files just in case.

You can also disable options like: - “Viewers and commenters can see the option to download, print, and copy” — but honestly, if they can see it, they can usually find a way to copy it anyway. (There’s no 100% foolproof way to prevent download once viewing is allowed.)

✨ Practical Tip¶

If you’re putting up your VBA tools + Help docs: - Zip the folder (if multiple files must stay together), - Upload the .zip, - Share the zipped file — easier and tidier for users to download in one go!

🌸 In short:¶

If you share a folder link, they can open and download the files.
If you want view-only safety, use “Viewer” permission and make sure you don’t accidentally set "Editor."

When you end a horn solo and wish the full section (or however many players are available) to resume playing in unison, the conventional directive is:

✅ "a2" (read: "a due") — meaning "for two" players¶

…or more generally: for all players on that staff to resume.

🎵 Here's how it's typically used:¶

Example progression:¶

  1. Horn I: Solo (only one player, usually principal)
  2. Later:
  3. Horn I resumes tutti playing → "a2"
  4. If both Horn I and II are playing on one staff, "a2" indicates both players play the same line in unison

If you have more than two horns, and each is on a separate staff, you’d simply resume writing in the appropriate staves without any further instruction.

✨ Additional Options (less common, but useful depending on layout):¶

Directive Meaning Use Case
"tutti" All players resume When multiple horns (or sections) were silent except for the soloist
"unis." Unison resumes (after divisi or solo) Often used in strings, but occasionally winds/brass if divisi was involved
"a4" / "a6" All 4 / 6 players resume Rare, but can be used in scores with large horn sections or unusual staff grouping

💡 Good Practice¶

  • Use "a2" if you've previously had a solo horn or divisi writing on a single staff.
  • If parts are written on separate staves, the return to tutti can be implicit from the scoring.
  • If there might be ambiguity, write "tutti" above the staff for safety.

1 beam Eighth notes (quavers)
2 beams Sixteenth notes (semiquavers)

🔍 What does Global = True actually mean?¶

Setting .Global = True tells the regex engine to find all matches of the pattern in the input string, not just the first.

Setting .Global = False tells it to stop after the first match.

🪄 When to use .Global = True¶

Use Global = True when:

  1. You want to extract all instances of a pattern — e.g., all dates, all dollar amounts, or every email address in a string.

  2. You're looping through all matches using something like: 

    For Each match In re.Execute(text)
    ' Process each match individually
Next

  3. You intend to collect matches into an array, or join them for analysis/reporting.

  4. You're doing pattern replacements and want to replace every occurrence.

🛑 When not to use .Global = True¶

Use Global = False when:

  1. You're only interested in the first match — this is common when parsing emails where you just want the first amount, or the first transaction ID.

  2. You’re relying on .Item(0) and using .SubMatches from just the first match (as you're doing in RgExEx).

  3. You only expect one valid match — for example, when parsing a structured field like: 

    Transaction ID: 9TW8473232972963G

Let me walk you through it in detail:

💡 What Happens During mkdocs serve¶

When you run:

mkdocs serve

you are not generating permanent HTML files.

Instead, MkDocs launches a temporary local development server, builds your Markdown files into HTML in memory, and serves them to your browser dynamically at http://127.0.0.1:8000.

This is useful for: - Previewing your site - Checking layout, navigation, and style - Testing links and media (images/videos) - Making real-time edits (with auto-refresh)

But no actual HTML files are written to disk during mkdocs serve.

🗂️ Where Are the Actual HTML Pages?¶

To generate the final static site, you must run:

mkdocs build

This: - Converts all your .md files to .html - Compiles assets like CSS and JS - Organizes everything into a single folder called:

/site

By default, on your Mac, that folder will appear in the same directory where your mkdocs.yml file lives. So if you're working in:

/Users/robyn/Desktop/Celeste/Art/

…then MkDocs will create:

/Users/robyn/Desktop/Celeste/Art/site/

That site/ folder will contain: - index.html - about/index.html (if you have an about.md) - assets/ (with your images, videos, stylesheets, etc.) - All the JavaScript and CSS required for the Material theme

🌐 Uploading to a Web Host (e.g., GoDaddy or Hostinger)¶

Yes, absolutely — the contents of that /site/ folder can be uploaded directly to any standard web host via: - FTP (using Cyberduck, FileZilla, etc.) - Control panel file manager (on cPanel) - WebDAV or Git deployment, if your host supports it

As long as your host supports plain HTML/CSS/JS (and most do), your site will run perfectly.

🔄 Refresh Rate (Hz) and Flicker¶

  • Refresh rate (measured in Hz) refers to how many times per second the screen redraws its image. For example:
  • 60Hz = 60 frames per second.

  • 180Hz = 180 frames per second.

  • Flicker, as experienced by the eye, can result from:

  • Low refresh rates (especially <60Hz).
  • PWM (Pulse Width Modulation) dimming — a method used to reduce brightness which can create invisible flicker detectable by sensitive users.

So: - A higher refresh rate can reduce perceived flicker, especially during motion (scrolling, video, etc.). - But if the monitor uses PWM, even a 180Hz screen can flicker, depending on its brightness setting and backlight tech.
